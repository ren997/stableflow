# Docs Overview

This directory contains the core product, design, and execution documents for StableFlow.

## Files

- `requirements.md`: Product goals, problem statement, MVP scope, and business requirements.
- `technical-design.md`: System architecture, module boundaries, payment flow, and core technical decisions.
- `implementation-guide.md`: Implementation-oriented guidance that translates design into backend development details.
- `dev-tasks.md`: Executable task list, task status, development phases, and recommended delivery order.

## Recommended Reading Order

1. `requirements.md`
2. `technical-design.md`
3. `implementation-guide.md`
4. `dev-tasks.md`

## Notes

- Read `requirements.md` first if you want product context.
- Read `technical-design.md` first if you want to understand the overall architecture.
- Read `dev-tasks.md` first only when you are already aligned on the product and design, and want to start coding immediately.
